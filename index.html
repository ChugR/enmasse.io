EnMasse

Messaging-as-a-Service

The EnMasse project provides the required tools and services for deploying and running a scalable messaging service on Openshift. Running on Openshift means that you can either run EnMasse on your own cluster locally or in the cloud. You can also run EnMasse on Openshift Origin, which is the upstream community project.

EnMasse uses the Qpid Dispatch Router, a high performance AMQP message router, in order to scale to a large number of clients and messages. For store-and-forward semantics, EnMasse uses message brokers such as Apache ActiveMQ Artemis.

EnMasse simplifies the setup and management of a messaging cluster by providing:

  • A simple configuration format encapsulating details of message brokers and routers. The configuration is split into a cluster admin config and application developer config.
  • A set of agents running as part of the cluster and acts on changes to the configuration
    • Configures the router network.
    • Generates and starts brokers for addresses marked as store-and-forward and attaches volumes for persistence.
  • Templates for quickly setting up a messaging cluster on Openshift.

The project is still in it's infancy, so there are currently a lot of TODO items that are being worked on in addition to the occasional bugfixing:

  • Supporting more message brokers (currently ActiveMQ and ActiveMQ Artemis only)
  • Supporting topics
  • Supporting durable subscriptions
  • Stabilizing configuration format
  • Downscaling brokers without loosing messages
  • Scalability and performance testing

To get started, you must have an openshift project setup. Use either Openshift Online to run it in the cloud, or Openshift Origin if you want to run the community edition yourself. Once you have a project setup, have a look at Setting up EnMasse for how to setup a messaging service.
