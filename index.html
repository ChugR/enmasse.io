EnMasse

Messaging-as-a-Service

EnMasse aims to provide the required tools and services for deploying and running a messaging service on Openshift. Running on Openshift means you can either run EnMasse on your own cluster locally or in the cloud. You can also run EnMasse on Openshift Origin, which is the upstream community project.

Some highlights of what EnMasse is:

  • A configuration format encapsulating details of brokers and routers
    • Separates cluster admin and application developer roles
  • A set of agents running as part of the cluster and acts on changes to the configuration
    • Configures the Qpid Dispatch Router cluster network
    • Generates and starts broker clusters for each address marked as store-and-forward and attaches volumes for persistence
  • Templates for quickly setting up a messaging cluster on Openshift

The project is still in it's infancy, so there are currently a lot of TODO items that are being worked on in addition to the occasional bugfixing:

  • Supporting other brokers (currently ActiveMQ and ActiveMQ Artemis only)
  • Supporting topics (only queues supported at the moment)
  • Stabilizing configuration format
  • Downscaling brokers without loosing messages
  • Scalability and performance testing
